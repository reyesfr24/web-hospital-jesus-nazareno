## 1. Planificación y definición del proyecto
- Elaborado un **Plan de Trabajo detallado** que cubre desde el inicio hasta el lanzamiento, dividido en fases claras: diseño, desarrollo, contenido, fotografía, pruebas, publicación y mantenimiento.
- Seleccionadas las tecnologías más adecuadas para garantizar que el sitio sea rápido, seguro, accesible para personas con discapacidad y fácil de actualizar en el futuro.
- Configurado el entorno de trabajo y el repositorio de código, que actúa como un sistema de guardado con historial de todos los cambios realizados.


## 2. Estructura base de código y navegación del sitio

Construido el "esqueleto" sobre el que se apoyan todas las páginas:

- Desarrollada la **cabecera del sitio** (parte superior).
- Desarrollado el **pie de página**.
- Preparadas las etiquetas SEO en cada página (título, descripción, datos para redes sociales) para que el sitio aparezca bien posicionado en buscadores como Google.


## 3. Desarrollo de las páginas principales

Construidas las estructuras de las páginas de contenido del sitio.

### Página de inicio
### Página de servicios
### Página del proceso de admisión
### Página de preguntas frecuentes (FAQ)
### Página de contacto
### Sección de noticias y actividades


## 4. Sistema de gestión de contenidos


Implementado un sistema sencillo para que en el futuro sea fácil añadir noticias y artículos al sitio sin necesidad de conocimientos técnicos avanzados:

- Las noticias se escriben en un formato de texto simple (Markdown), similar a escribir en Word.
- Definido un esquema para cada noticia: título, descripción, fecha, categoría, imagen y si es destacada o no.
- El sitio genera automáticamente la página de cada noticia a partir de ese archivo de texto.



## 5. Accesibilidad y optimización

Seguidas desde el inicio las mejores prácticas para que el sitio sea accesible y rápido:

- El sitio cumple con los estándares de accesibilidad **WCAG 2.1 nivel AA**, lo que significa que puede ser usado por personas con discapacidad visual o motriz (lectores de pantalla, navegación por teclado, etc.).
- Añadidas etiquetas descriptivas en todos los elementos interactivos para que las tecnologías de asistencia los interpreten correctamente.
- El sitio genera páginas en HTML estático, lo que lo hace extremadamente rápido de cargar.
- Preparadas las bases para obtener puntuaciones superiores a 90/100 en las métricas de Google (rendimiento, accesibilidad, buenas prácticas y SEO).
